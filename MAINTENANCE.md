# 日常维护说明

这个站点使用 VitePress 生成静态网页。日常维护时，你主要需要修改 Markdown 文件，然后把改动提交并推送到 Git 仓库；部署平台会自动重新发布。

## 如果我想改某一章的内容，我该做什么？

1. 在仓库里找到对应的 `.md` 文件，例如 `ch5-subordination.md`。
2. 直接修改文字、例句或表格。
3. 用 Git 提交这次改动，提交信息可以写成“更新第 5 章从属内容”。
4. 推送到远程仓库。
5. 等待 GitHub Pages 或 Netlify 自动构建完成，通常几十秒到几分钟后网站就会更新。

## 如果我想新增一章，我该做什么？

1. 新建一个英文文件名的 Markdown 文件，例如 `ch8-revision.md`。
2. 在 `.vitepress/config.mjs` 里的 `guide` 列表中加入这一章的标题和链接，例如：

   ```js
   { text: '第 8 章 修改与润色', link: '/ch8-revision' }
   ```

3. 如果其他章节需要跳转到新章节，使用不带 `.md` 后缀的链接，例如 `[第 8 章](ch8-revision)`。
4. 提交并推送改动，等待自动发布。

## 如果站点出了问题，我该查看哪里？

1. 如果页面内容不对，先检查对应的 `.md` 文件。
2. 如果目录、上一章/下一章或页面顺序不对，检查 `.vitepress/config.mjs` 里的 `guide` 列表。
3. 如果线上没有更新，去部署平台查看构建日志：
   - GitHub Pages：仓库页面的 `Actions` 标签页。
   - Netlify：项目页面的 `Deploys` 标签页。
4. 如果提示链接错误，检查 Markdown 中的链接目标是否和文件名一致，并且不要带 `.md` 后缀。

## 本地预览

如果你愿意使用一点命令行，可以先安装 Node.js 22 LTS，然后在仓库目录运行：

```bash
npm install
npm run dev
```

终端会显示一个本地地址，通常是 `http://localhost:5173/`。用浏览器打开它，就能在发布前预览站点。
